Error Taxonomy (Observed Failures):

  • Missing fields (~25%): Expected keys absent from output
  • Inconsistent key names (~15%): Keys with wrong naming (e.g., person_name vs name)
  • Type mismatches (~20%): Booleans as strings ("true" vs true)
  • Extra fields (~10%): Additional keys not in schema
  • Truncated outputs (~5%): JSON cut off mid-structure

🔧 Prompt Engineering Iteration

Initial Prompts → Final Optimized Version

❌ Version 1 (Problem):

Extract information to JSON format.

Result: 63% validity - Too vague, missing schema, inconsistent output

⚠️ Version 2 (Partial Fix):

Convert to valid JSON. Return JSON only. Schema: {schema}

Result: 78% validity - Better but still inconsistent, occasional truncation

✅ Version 3 (Final):

You are a JSON expert. Your task is: 1. Extract exactly the keys requested in the schema 2. Return ONLY valid JSON (no explanations, no code blocks) 3. Match the schema exactly 4. Use proper data types: string, number, boolean, array Schema: {schema} Input: {input} Return only valid JSON:

Result: 95% validity ✅ - Significantly improved consistency and schema compliance

💡 Key Takeaways

Sequential fine-tuning is effective for teaching structured output generation.

While a small amount of forgetting occurs after Stage 2 training, the model successfully balances:

  • ✅ General instruction-following capabilities (Stage 1: 0.61 win rate)
  • ✅ Structured JSON output generation (Stage 2: 0.938 validity)
  • ✅ Trade-off optimization through hyperparameter tuning (LR 1e-5 best)
  • ✅ No catastrophic forgetting (mild 18.5% average degradation)

Complete evaluation includes per-category forgetting analysis, error taxonomy, field-level F1 scores, and prompt iteration history showing the effectiveness of sequential fine-tuning when carefully designed.

📊 This project demonstrates:

  • Balanced approach to specialized learning and generalization
  • Importance of held-out evaluation sets for fair assessment
  • Value of prompt engineering iteration (v1→95% validity)
  • Practical constraints of fine-tuning trade-offs
